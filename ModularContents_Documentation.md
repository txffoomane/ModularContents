# Полное руководство по созданию контента для ModularContents

Мод **ModularContents** (версия для Forge 1.12.2) — это мощный инструмент для создания собственных предметов, рецептов, лут-таблиц, креатив-вкладок и механик аирдропов без знания языка Java. Весь контент настраивается через конфигурационные JSON-файлы или с помощью внутриигрового визуального редактора.

---

## 1. Структура Контент-Пака

Все ваши контент-паки хранятся в корневой директории игры в папке `ModularContents`.
Вы можете создавать неограниченное количество паков. Каждый пак — это отдельная папка.

**Пример правильной структуры:**
```text
.minecraft/
 └── ModularContents/
      └── MyAwesomePack/
           ├── items/                   <- Кастомные предметы (.json)
           ├── recipes/                 <- Рецепты для верстака (.json)
           ├── tabs/                    <- Свои вкладки креатива (.json)
           ├── loot_tables/
           │    ├── airdrops/           <- Таблицы лута для аирдропов (.json)
           │    └── equipment/          <- Пресеты лута для сундуков (.json)
           └── assets/                  <- Текстуры, модели и языковые файлы (как в обычных ресурс-паках)
```

## 2. Внутриигровой Генератор (Content Creator)

Для быстрого старта вам не обязательно писать код вручную. Зайдите в игру в **Творческом режиме (Creative Mode)** и нажмите клавишу **F8** (можно изменить в настройках управления).

Откроется интерфейс, позволяющий визуально генерировать JSON-файлы:
- **Loot:** Положите предметы в слоты, кликните по ним, чтобы задать минимальное/максимальное количество и шанс выпадения. Нажмите "Generate JSON".
- **Items:** Введите название, прочность, размер стака и ID вкладки. Мод сам сгенерирует JSON предмета.
- **Recipes:** Положите требуемые ресурсы (Inputs) в нижние слоты, а желаемый результат (Outputs) — в верхние. Задайте время крафта и нажмите "Generate".
- **Tabs:** Впишите название вкладки и предмет-иконку (например, `minecraft:apple`).

Все сгенерированные файлы сохраняются в папку: `ModularContents/generated/`. Оттуда их нужно переместить в папку вашего пака (например, в `MyAwesomePack/`).

## 3. Создание Предметов (Items)

Если вы хотите писать контент вручную или узнать все возможные параметры, создайте файл `имя_предмета.json` в папке `items/`.

**Полный пример предмета:**
```json
{
  "id": "vip_radio",
  "display_name": "VIP Radio",
  "max_stack_size": 1,
  "creative_tab": "my_custom_tab",
  "max_damage": 5,
  "airdrop_caller": {
    "is_flare": false,
    "delay_min": 100,
    "delay_max": 200,
    "distance_min": 10.0,
    "distance_max": 50.0,
    "loot_table": "military_loot",
    "consume_on_use": true
  }
}
```

### Параметры предмета:
*   `id` (Обязательно): Внутреннее имя предмета. В игре он будет называться `modularcontents:vip_radio`.
*   `display_name`: Название предмета, которое увидят игроки в инвентаре.
*   `max_stack_size`: Максимальное количество предметов в слоте (от 1 до 64).
*   `creative_tab`: ID вкладки креатива, в которой появится предмет. Можно использовать ванильные (например, `misc`, `combat`, `tools`) или свои кастомные.
*   `max_damage`: Прочность предмета. Если `0` — предмет не ломается. Если больше `0` (например, `5`) — предмет можно использовать 5 раз, после чего он сломается.

### Блок `airdrop_caller` (Опционально)
Превращает этот предмет в вызыватель аирдропов (как рация или сигнальная ракета):
*   `is_flare`: Если `true`, предмет кидается как граната и испускает цветной дым на месте приземления. Если `false`, аирдроп вызывается мгновенно кликом ПКМ.
*   `delay_min` / `delay_max`: Задержка перед падением аирдропа в тиках (20 тиков = 1 секунда).
*   `distance_min` / `distance_max`: Расстояние от игрока, на котором заспавнится аирдроп (для `is_flare: false`).
*   `loot_table`: Название конкретной JSON таблицы лута, которую вызовет предмет. Если оставить `""`, таблица выберется случайно.
*   `consume_on_use`: Тратится ли предмет (или прочность предмета) при использовании.

## 4. Создание Рецептов (Recipes)

Рецепты для Кастомного Верстака сохраняются в формате JSON в папке `recipes/`. Верстак поддерживает вероятностный крафт (шанс получить предмет).

**Полный пример рецепта:**
```json
{
  "id": "craft_radio",
  "category": "electronics",
  "craftingTime": 400,
  "minDrops": 1,
  "inputs": [
    {
      "item": "minecraft:iron_ingot",
      "count": 5,
      "meta": 0
    },
    {
      "item": "minecraft:redstone",
      "count": 2
    }
  ],
  "outputs": [
    {
      "item": "modularcontents:radio",
      "count": 1,
      "chance": 100.0
    },
    {
      "item": "minecraft:gold_nugget",
      "count": 1,
      "chance": 25.0
    }
  ]
}
```

### Параметры рецепта:
*   `id`: Уникальное имя рецепта.
*   `category`: Категория в интерфейсе верстака (сортировка по вкладкам слева в верстаке).
*   `craftingTime`: Время крафта в тиках (20 тиков = 1 секунда. 400 = 20 секунд).
*   `minDrops`: Минимальное количество предметов, которое игрок гарантированно получит, если у всех `outputs` не сработает шанс.
*   `inputs`: Массив требуемых предметов. Указывается `item` (ID), `count` (количество) и `meta` (опционально, подтип блока/предмета).
*   `outputs`: Массив результатов крафта. Можно указать `chance` (от 0.0 до 100.0), чтобы предмет выпадал с определенной вероятностью (например, 25% шанс получить золотой самородок как побочный продукт).

## 5. Создание Вкладок (Creative Tabs)

Чтобы ваши предметы не путались с ванильными, создайте свои вкладки в папке `tabs/`.

**Пример:** `my_cool_tab.json`
```json
{
  "id": "my_cool_tab",
  "display_name": "My Awesome Mod",
  "icon": "minecraft:diamond_sword"
}
```
*   `id`: Внутреннее имя вкладки. Именно его нужно писать в `creative_tab` в конфигурации предметов.
*   `display_name`: Имя вкладки в игре.
*   `icon`: Предмет, который будет отображаться на иконке вкладки (можно указать свой предмет: `modularcontents:my_custom_item`).

## 6. Аирдропы и Лут-таблицы (Loot Tables)

Система аирдропов генерирует лут динамически. Таблицы лежат в `loot_tables/airdrops/`.

**Пример:** `military_loot.json`
```json
{
  "weight": 30,
  "items": [
    {
      "item": "minecraft:iron_sword",
      "meta": 0,
      "min": 1,
      "max": 1,
      "chance": 0.5
    },
    {
      "item": "minecraft:arrow",
      "min": 16,
      "max": 32,
      "chance": 0.8
    }
  ]
}
```
*   `weight`: Вес таблицы (например, 30). При случайном спавне аирдропа система складывает веса всех таблиц и бросает кубик. Чем выше вес, тем чаще падает именно этот аирдроп.
*   `items`: Список предметов.
    *   `chance`: Шанс появления предмета в сундуке (0.5 = 50%, 1.0 = 100% гарантия).
    *   `min` и `max`: Если предмет заспавнился, его количество в стаке будет случайным в этом диапазоне (например, от 16 до 32 стрел).

### Способы вызова аирдропов:
1.  **Глобальный спавн:** Настраивается в `ModularContents/modularcontents_settings.cfg`. Мод будет сам скидывать дропы раз в заданное количество игровых дней случайному игроку на сервере.
2.  **Предметы призыва:** Базовые `modularcontents:radio` и `modularcontents:signal_flare` или ваши кастомные.
3.  **Тактический Ноутбук (`modularcontents:laptop`):** Установите блок, кликните по нему ПКМ. Откроется радар-миникарта местности 256x256. Кликните мышкой на карту, чтобы выбрать точную точку падения груза.
4.  **Команда:** `/mc airdrop <x> <z> [имя_таблицы]`

## 7. Текстуры и Модели

Мод `ModularContents` включает в себя встроенный загрузчик ресурс-паков. Вы можете изменять текстуры своих предметов без сторонних модов!

Внутри вашего контент-пака создайте папку `assets`:
```text
MyAwesomePack/
 └── assets/
      └── modularcontents/
           ├── textures/
           │    └── items/
           │         └── vip_radio.png    <- Картинка 16x16
           └── models/
                └── item/
                     └── vip_radio.json   <- Ванильная модель предмета
```

**Пример `vip_radio.json` (стандартный плоский предмет):**
```json
{
  "parent": "item/generated",
  "textures": {
    "layer0": "modularcontents:items/vip_radio"
  }
}
```

---

## 8. Полезные команды

*   `/mc reload` — Мгновенно перезагружает все рецепты, лут-таблицы и предметы. Вам не нужно перезапускать сервер или клиент, чтобы протестировать новый контент (команда требует права Оператора/OP).
*   `/mc airdrop <x> <z> [loot_table]` — Принудительный вызов аирдропа администратором.

## 9. Заполнение Сундуков (Zone Equipment / Chest Loot)

Мод позволяет динамически заполнять сундуки (и любые другие контейнеры с инвентарем) случайным лутом по выделенной зоне. Это настраивается через **Equipment Manager** и пресеты лута.

### 9.1 Создание пресетов (Equipment JSON)
Пресеты лута для контейнеров сохраняются в папку `loot_tables/equipment/` в формате JSON. Каждый пресет может содержать несколько "наборов" (sets) с шансом выпадения (chance). Если сумма шансов наборов меньше 100%, возможна ситуация, когда не выпадет ни один набор.

**Пример:** `example_equipment.json`
```json
{
  "sets": [
    {
      "name": "rare_loot",
      "chance": 10.0,
      "items": [
        {
          "item": "minecraft:diamond",
          "count": 1,
          "meta": 0
        }
      ]
    },
    {
      "name": "basic_loot",
      "chance": 90.0,
      "items": [
        {
          "item": "minecraft:iron_ingot",
          "count": 3,
          "meta": 0
        },
        {
          "item": "modularcontents:example_item",
          "count": 1,
          "meta": 0
        }
      ]
    }
  ]
}
```

### 9.2 Использование GUI заполнения (Zone Equipment)
1. Выдайте себе или скрафтите предмет **Equipment Manager** (или привяжите его к административной панели).
2. При использовании откроется карта местности (радар 256x256). На карте желтыми точками отмечены все контейнеры (сундуки, печки, воронки и т.д.).
3. **Выделение зоны:** Зажмите Левую Кнопку Мыши на карте и выделите область с сундуками.
4. **Выбор пресетов:** Справа появится список всех загруженных пресетов из `loot_tables/equipment/`. Включите нужные пресеты (галочкой) и настройте ползунок шанса (с какой вероятностью сундук в зоне получит этот пресет).
5. \*\*Fill (Авто-зона):\*\* Нажмите `Fill`, чтобы заполнить сундуки лутом и создать зону респавна. После того как сундук опустеет, лут появится снова (через 5 мин). Зона сохраняется в мире.
6. \*\*Clear:\*\* Нажмите `Clear`, чтобы очистить сундуки и удалить зону респавна с этих координат.
